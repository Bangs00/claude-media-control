# ステータスラインに再生中の曲を出す

[English](statusline.md) | [한국어](statusline.ko.md) | **日本語** | [简体中文](statusline.zh-CN.md)

Claude Code のステータスラインに、いま再生している曲を 1 行追加できます:

```
[既存のステータスラインはそのまま]
▶︎ Karma Police — Radiohead  ━━━━━━────  2:13/4:24
```

この行を作るのは `media.sh statusline` です。小さな TTL キャッシュ
（デフォルト 1 秒）から 50ms を大きく下回る速さで応答するので、ステータス
ラインが遅くなることはありません。実際の再生情報の読み取りは TTL の区間ごと
に最大 1 回だけ。そのためステータスラインが再描画されるたび、経過時間と
プログレスバーはおよそ 1 秒刻みで進んでいきます。

## 有効にする

Claude Code の中で:

```
/media:config display.statusline on
```

セットアップはこれだけです。有効化のときはまず、再生情報を実際に
読み取れるかを検証し（拒否された場合は `/media:doctor` を実行してみて
ください）、続いてセグメントを Claude Code に自動で配線します:

1. `~/.claude/settings.json` にある現在の `"statusLine"` の値を
   `~/.claude/statusline-media.backup.json` にバックアップします
   （もともとなかった場合は `null`）。
2. `~/.claude/statusline-media.sh` にラッパースクリプトを生成します。
   既存のステータスラインコマンドをまず実行し、そのあとに再生情報の行を
   追加するスクリプトです。
3. `settings.json` の `statusLine` がラッパーを指すようにします。エントリの
   ほかのキー（例: `padding`）はすべて保持され、自分で設定していなければ
   `refreshInterval: 1` が追加されます — ステータスラインは本来、会話
   イベントのときにしか更新されず、この 1 秒ごとの再実行があるからこそ、
   何もしていない間も経過時間とプログレスバーが動きます（再描画を
   減らしたければ `settings.json` で値を上げるか外してください。再描画の
   たびに既存のステータスラインコマンドも一緒に実行されます）。

セグメントは次のステータスラインの更新で現れます — 再起動も手作業の手順も
不要です。`/media:statusline` でセグメントを配置したときも、同じように
有効化（と配線）されます。

## クリックで操作する

ハイパーリンク対応のターミナルでは、セグメントが **⌘+クリック**に反応
します:

| 対象 | ⌘+クリックの動作 |
| --- | --- |
| `▶︎` / `⏸` アイコン | 再生 / 一時停止の切り替え |
| タイトル — アーティスト（と `(アプリ)`） | 再生中のアプリを最前面に |
| プログレスバー | シーク — 10 個のセルそれぞれがその位置へ（5%、15%、… 95%） |

仕組み: 各パーツはローカルの `claude-media://` URL スキームを指す OSC 8
ハイパーリンクで包まれます。ステータスラインを有効にすると、小さな
ハンドラアプリ（`ClaudeMediaClick.app` — macOS 標準の `osacompile` で
プラグインのデータディレクトリに生成、サードパーティコードなし）が作られ、
LaunchServices に登録されます。クリックすると `media.sh open-url` が実行
され、受け付ける操作は toggle・activate・パーセントシークのちょうど 3 つ
だけで、それ以外はすべて拒否します。ブラウザ再生では Web コンテンツの
ヘルパーを所有アプリに解決して最前面化します（例: `com.openai.atlas.web`
→ ChatGPT Atlas）。

補足:

- **ターミナル対応**: iTerm2、Ghostty、WezTerm、Kitty、VS Code、Alacritty
  0.11+ が OSC 8 リンクをクリック可能にします（通常は ⌘+クリック。カスタム
  URL スキームは初回に確認を求めることがあります）。ハイパーリンク非対応の
  ターミナル（例: Terminal.app）ではリンクのバイト列は見えないため、ただの
  通常セグメントとして表示されます。tmux 3.4+ はハイパーリンクを素通し
  します。
- クリック後、次の更新（1 秒以内）でセグメントに反映されます: アイコンが
  切り替わり、バーがジャンプします。
- スイッチは `statusline.links`（デフォルト `on`）です:
  `/media:config statusline.links off` でリンクなしの通常セグメントに
  戻り、再びオンにするとハンドラアプリを（再）生成・登録します。その
  ビルドに失敗した場合は**拒否されます（exit 3）**。リンクはハンドラ
  アプリが実際に存在する間だけ描画されます — 誰も応えないリンクは、ない
  よりも悪いからです。`/media:doctor` がハンドラの状態を報告します
  （`Click links`）。
- URL スキームはその性質上システム全体のものです — どのアプリでも
  `claude-media://` URL を開けます。ハンドラが公開する表面は再生 /
  一時停止・アプリの最前面化・シークのみで、最悪でも迷惑レベル、
  キーボードのメディアキーと同じ部類です。
- プラグインをアンインストールする（または `media.sh statusline
  uninstall`）と、ハンドラアプリもほかの配線と一緒に登録解除・削除
  されます。

## 設計上の保証（安心して任せられる理由）

1. 既存のステータスラインコマンドは**置き換えません** — ラッパーがまず
   それを元のまま実行し、その出力は**1 バイトも変えずに**素通しします。
   再生情報は必ず**独立した 1 行として追加されるだけ**です。
2. `display.statusline` がオフのとき（デフォルト）、セグメントは何も
   出力しません。空行すら出しません。Claude Code は存在しない行を詰めて
   くれるので、ステータスラインの見た目は以前とまったく同じです。
   （`off` はセグメントを即座に隠すだけで配線は残すので、再びオンにする
   のも一瞬です。）
3. `settings.json` で触れるキーは `statusLine` のただ 1 つで、それも必ず
   以前の値を `statusline-media.backup.json` に保存してからです。
   書き込みはアトミックで、シンボリックリンクをたどり（dotfile 構成でも
   安全です）、ほかの設定キーには一切触れません。
4. **プラグインをアンインストールすると、すべてが自動で元に戻ります。**
   Claude Code にはプラグインが使えるアンインストールフックがないため、
   ラッパーは自己修復型です: ステータスラインが更新されるたびに
   インストール済みプラグインの一覧を確認し、プラグインが消えていたら、
   バックアップしておいた `statusLine` を `settings.json` に復元して、
   自分自身とバックアップファイルを削除し、`claude-media://` クリック
   ハンドラアプリも同じように登録解除して消します。何も残りません —
   アンインストールから 1 秒以内に、ステータスラインは元の姿そのままに
   戻ります。
5. プラグインを**無効化**しているだけの間、ラッパーは何も追加せずに
   待ちます — 以前のステータスラインはいつもどおり動き、配線は再び
   オンにするときのために残ります。
6. **手作業で**配線したステータスライン（下のレシピや、セグメントを
   すでに実行している任意のコマンド）は検出され、インストール時も
   アンインストール時も決して変更されません。

プラグインをアンインストールせずに配線だけ外すには — バックアップを
復元し、ラッパーとバックアップファイルを削除して、`display.statusline`
もオフにします:

```
media.sh statusline uninstall     # または Claude に「ステータスラインの配線を外して」と頼むだけ
```

`media.sh statusline status` は現在の配線状態（`managed`、`manual`、
`none`）を報告し、`/media:doctor` のレポートにも含まれます。

## 項目を配置する

`/media:statusline` を実行してください — セグメントの見た目をまとめて決める
ハブです。タブが 3 つ開きます: **Items**（音量・プログレスバー・時間・出力
デバイスのオン / オフ）、**Layout**（Standard / Stacked または数字
パターン）、**Style**（パーツごとのスタイル — 次の節）。パターンは下の凡例で
組みます:

| # | 項目 | 表示例 |
| --- | --- | --- |
| 1 | `track` | `▶︎ Karma Police — Radiohead` |
| 2 | `app` | `(Spotify)` |
| 3 | `volume` | `🔉 ▄ 45%` — アイコン + 音量に応じた高さのバー + パーセント。ミュート中は `🔇` |
| 4 | `progressbar` | `━━━━━━────` |
| 5 | `time` | `2:13/4:24` |
| 6 | `output` | `🎧 AirPods Pro` — アイコンはデバイス種別に追従: `🎧` Bluetooth・ヘッドホンジャック · `📺` HDMI/DisplayPort · `📶` AirPlay · `🔊` スピーカー |

数字の並びがそのまま表示順です。`/` で新しい行が始まります。書かなかった
数字の項目は表示されません。たとえば `123/456` なら 1 行目に track・
アプリ・音量、2 行目に残りが並びます。デフォルトの構成は
`track app progressbar time` で、オン / オフのクイック切り替えと
ステータスライン全体のリセットは `/media:config` にあります。

レイアウトの挙動:

- **順序** — 項目は保存した順序どおりに描画されます。「時間を先頭に」と
  頼んでもいいし、リストを直接指定もできます:
  `/media:config statusline.fields "time,progressbar,track,app"`。
- **行単位の明示レイアウト** — 項目リストに `/` を入れると、そこで行が
  変わります。各行にはそこに置いた項目だけがその順序で表示されます。
  表示するものがない行は行ごと消えます（例: ネイティブヘルパーがない
  ときの `output`）。
- **グループレイアウト**（リストに `/` がないとき）— 1 行にまとめるか、
  `statusline.multiline on` でグループごとに行を分けます。グループの規則:
  `app` は track に付きます。`progressbar` と `time` は隣り合うと 1 つの
  グループになります。`output` と `volume` は隣り合う track グループに
  合流し、両者が隣り合えば 2 つで 1 つのグループになります。

Standard — 全項目を 1 行で（パターン `123456`）:

```
▶︎ Karma Police — Radiohead (Spotify)  🔉 ▄ 45%  ━━━━━━────  2:13/4:24  🎧 AirPods Pro
```

Stacked — 明示的な 2 行レイアウト（パターン `123/456`、つまり
`statusline.fields "track,app,volume,/,progressbar,time,output"`）:

```
▶︎ Karma Police — Radiohead (Spotify)  🔉 ▄ 45%
━━━━━━────  2:13/4:24  🎧 AirPods Pro
```

出力デバイスを track の行に、音量は外して（パターン `126/45`）:

```
▶︎ Karma Police — Radiohead (Spotify)  🎧 AirPods Pro
━━━━━━────  2:13/4:24
```

時間を先頭に、1 行で（パターン `5412`、つまり
`statusline.fields "time,progressbar,track,app"`）:

```
2:13/4:24  ━━━━━━────  ▶︎ Karma Police — Radiohead (Spotify)
```

`output` と `volume` の項目にはネイティブヘルパーが必要です（セグメントが
元々行う読み取りに相乗りするため、追加コストはありません）。デバイスの
切り替えは `/media:output`、音量の変更は `/media:volume` で行え、セグメント
には次の更新で反映されます。

### 長いタイトル: マーキースクロール

30 セル（ターミナルの桁数）を超えるタイトルは、固定幅 30 セルの窓の中を
1 秒に 1 文字ずつ流れていきます。（窓は再描画のたびに 1 文字進みます —
下記の 1 秒リフレッシュを参照。）

```
▶︎ ing Willow (10 Minute Version)  — Taylor Swift (Music)
```

漢字・かな・ハングルは 2 セルとして数えるので、CJK のタイトルでも窓の幅は
一定に保たれます。どんなに長くてもタイトル全体を表示したい場合はオフに
できます:

```
/media:config statusline.marquee off
```

### 色とパーツごとのスタイル

セグメントはデフォルトでスタイル付きです。Claude Code のステータスラインは
ANSI コードをレンダリングでき、ラッパーはそれを手を加えずに通します:

- ▶︎/⏸ アイコン、プログレスバーの塗りつぶし部分、音量バーは、再生状態に
  合わせて色が変わります（再生中は green、一時停止中は yellow）
- **太字**の曲名と経過時間（動き続ける部分なのでくっきり見えます）、
  *斜体*のアーティスト、薄い表示の合計時間・空きセル・アプリ名・出力デバイス

使うのは標準の 16 色 SGR コードだけなので、実際の色味はターミナル自身の
パレットに従います。プレーンテキストがよければ
`/media:config statusline.color off` を実行してください。`NO_COLOR`
環境変数も有効です。

さらに、**どのパーツも個別にスタイリングできます**。`/media:statusline` の
Style タブを使うか、希望をそのまま言葉で伝えるか（「曲名を太字のシアンに」
「バーのスタイルを dots に」「音量アイコンを ♪ に」「アーティストを隠して」）、
キーを直接設定してください。テキスト系のキーは `bold dim italic underline`
の組み合わせに色を 1 つ（`black red green yellow blue magenta cyan white`
または `bright-<色>`）、`none`（スタイルなし）、あるいは **`off`（その
パーツを非表示）** を受け付けます:

| キー | 対象 | デフォルト |
| --- | --- | --- |
| `style.track.title` / `style.track.artist` | 曲名 / アーティスト | `bold` / `italic` |
| `style.app` | アプリ名 `(Spotify)` | `dim` |
| `style.time.elapsed` / `style.time.total` | `2:13` / `/4:24` | `bold` / `dim` |
| `style.volume.icon` / `style.volume.style` / `style.volume.bar` / `style.volume.percent` | 音量アイコン / バーの形 / バーの表示 / パーセント | `auto` / `block` / `on` / `dim` |
| `style.progressbar.playing` / `style.progressbar.paused` | バーの塗り + ▶︎/⏸ のアクセント | `green` / `yellow` |
| `style.progressbar.style` | プログレスバーの文字 | `line` |
| `style.output.icon` / `style.output` | 出力アイコン / デバイス名 | `auto` / `dim` |

非表示は周りも一緒に片づけます: 曲名を隠せば `—` の区切りも消え、経過時間を
隠せば合計時間の前の `/` も消えます。パーツが全部隠れた項目は項目ごと
消えます。（項目を丸ごと外すのは配置の仕事です — パターンからその数字を
抜いてください。）

プログレスバーの文字は `style.progressbar.style` で決まります:

| プリセット | 見た目 | |
|---|---|---|
| `line`（デフォルト） | `━━━━━━────` | |
| `blocks` | `██████░░░░` | |
| `smooth` | `█████▋░░░░` | 境界のマスが ⅛ 刻みの部分ブロック |
| `knob` | `━━━━━●────` | スライダーのつまみが先端に付く |
| `wave` | `▂▄▆▄▂▄▁▁▁▁` | 波 — 再生中に流れる |
| `pulse` | `▂▂█▁▄▂▁▁▁▁` | 心電図の拍動 — 再生中に流れる |
| `eq` | `▂▇▃█▅▆▁▁▁▁` | イコライザー — 再生中に流れる |
| `notes` | `♪♫♪♫♪♫····` | 音符 — 再生中に行進する |
| `braille` | `⣿⣿⣿⣿⣿⣿⣀⣀⣀⣀` | |
| `chevron` | `▸▸▸▸▸▸▹▹▹▹` | |
| `tape` | `▰▰▰▰▰▰▱▱▱▱` | |
| `cassette` | `▮▮▮▮▮▮▯▯▯▯` | |
| `retro` | `======----` | 純粋な ASCII |
| `dots` | `●●●●●●○○○○` | |

「塗り + 空き」を意味する任意の 2 文字も使えます（`"#-"` →
`######----`）。動くプリセットは再生中に波形が毎秒空き側へ流れ、一時停止で
止まります。`/media:now` の返信に出るバーも同じ文字で描かれるため、
2 つの表示は常に一致します。音量バーの形は `style.volume.style` です:
`block`（音量に応じて高さが変わる `▄` 1 つ、デフォルト）、
`progress`（プログレスバーの文字で描く 5 マスのミニバー）、
`stairs`（`▂▄▆█` の階段）。形がどれでも、音量バーの色はプログレスバーの
再生 / 一時停止色に従います — セグメント全体でアクセントは 1 つ。
`style.volume.bar` はバーのオン / オフを切り替えるだけのスイッチです
（デフォルト `on`）。アイコン（`style.volume.icon`、
`style.output.icon`）は `auto`（レベル別 / デバイス種別）、`none`（非表示）、
または `♪` のような任意のグリフで、ミュート中は常に 🔇 が表示されます。
文字を変えるキーと `off` は色をオフにしていても効きます。それ以外のキーは
`statusline.color` がオンのときに反映されます。

```
/media:config style.track.title "bold cyan"    # 1 パーツだけ設定
/media:config style.track.title reset          # そのパーツだけデフォルトに
/media:config style reset                      # スタイルを全部デフォルトに
/media:config statusline reset                 # 配置・行・色・マーキー・
                                               # スタイルまで丸ごと初期状態に
```

`media.sh config style` を実行すると、全キーの現在値とデフォルトが一覧で
出ます。変更は次のステータスラインの更新で即座に反映され、再起動は不要です。

## 手動セットアップ（カスタムステータスライン）

配線を自分で管理したい場合 — たとえばセグメントを行として追加するのでは
なく、自分のステータスラインスクリプトの*中に*埋め込みたい、という場合は？
コマンドを**先に**用意してから、セグメントを有効にしてください。
自動配線は、セグメントをすでに実行している `statusLine` コマンド
（`statusline-media.sh` か `media.sh … statusline` を含むもの）を認識して
完全にそのままにするので、有効化は表示のトグルを切り替えるだけになります。

出発点になる汎用ラッパー — `~/.claude/statusline-media.sh` として保存し、
実行権限を付けてください（`chmod +x ~/.claude/statusline-media.sh`）:

```bash
#!/bin/bash
# statusline-media.sh — 既存のステータスライン（そのまま）+ 再生情報の行。
input=$(cat)

# ── 1. 既存の statusLine コマンドを、引用符の間にそのまま貼り付けます。
#       settings.json の "statusLine" 配下にある "command" の値を使って
#       ください。もともとステータスラインがなかった場合は EXISTING を
#       空のままにします。
EXISTING=''
if [ -n "$EXISTING" ]; then
  printf '%s' "$input" | bash -c "$EXISTING"
fi

# ── 2. 再生情報（オフのとき / 何も再生していないとき / プラグインが
#       ないときは何も出力しません）。実行のたびにインストール済みの
#       最新バージョンを探すので、プラグインを更新してもラッパーは
#       そのまま動きます。
MEDIA_DIR="$(ls -d "$HOME"/.claude/plugins/cache/claude-media-control/media/*/ 2>/dev/null \
  | awk -F/ '{ print $(NF-1) "\t" $0 }' \
  | sort -t. -k1,1n -k2,2n -k3,3n \
  | tail -1 | cut -f2-)"
if [ -n "$MEDIA_DIR" ] && [ -x "${MEDIA_DIR}scripts/media.sh" ]; then
  np="$("${MEDIA_DIR}scripts/media.sh" statusline 2>/dev/null)"
  [ -n "$np" ] && printf '%s\n' "$np"
fi
exit 0
```

その上で、`~/.claude/settings.json` を自分でこのファイルに向けます:

```json
{
  "statusLine": {
    "type": "command",
    "command": "\"$HOME/.claude/statusline-media.sh\"",
    "refreshInterval": 1
  }
}
```

（`refreshInterval: 1` は、何もしていない間も時間とバーを動かし続ける
ための指定です — 上の「有効にする」を参照。）
チェックアウトから開発している場合（`claude --plugin-dir`）は、`MEDIA_DIR`
のブロックをリポジトリのパスに置き換えてください:
`np="$(/path/to/claude-media-control/scripts/media.sh statusline 2>/dev/null)"`

## メンテナンスのヒント

- **自動配線（managed）**: ラッパーは生成されるファイルです — 直接編集
  しないでください。プラグインの更新時（セッション開始時のウォームアップ）
  と `media.sh statusline install` の再実行時に作り直されます。
  `media.sh statusline uninstall` は配線を外して以前のステータスラインを
  復元します。プラグインをアンインストールした場合も、次のステータス
  ラインの更新で同じことが自動で行われます。
- **手動配線（manual）**: ファイルはユーザーのもので、プラグインは決して
  触りません。あとでステータスラインの構成を変えたら、`EXISTING` の行も
  忘れずに更新してください。元に戻すには、`settings.json` の
  `"statusLine"` を元の値に戻し、自分のラッパーを削除します。プラグインを
  アンインストールするだけでもセグメントは自然に消えます（プラグインの
  設定はデータディレクトリごと消えます）が、ファイル自体は自分で消して
  ください。
- `/media:config display.statusline off` は即座に効きます。オフにした瞬間に
  キャッシュ済みの行が削除されるので、ステータスラインの再起動は不要です。
