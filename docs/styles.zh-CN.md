# 状态栏样式图鉴

[English](styles.md) | [한국어](styles.ko.md) | [日本語](styles.ja.md) | **简体中文**

正在播放组件里每一处看得见的细节，都对应一个配置键。这一页把**所有可用的
样式连同实际效果**一次列全。改法有两种：

```
/media:statusline                              # 交互式——或者直接开口：
                                               #   "进度条换成 dots"、"把歌手隐藏"
/media:config style.progressbar.style wave     # 直接设置某个键
```

改动会在下一次状态栏刷新（1 秒内）立即生效，永远不需要重启。运行
`media.sh config style` 可以列出每个键的当前值和默认值。

## 组件解剖图

```
▶︎ Karma Police — Radiohead (Spotify)  🔉 ▄ 45%  ━━━━━━━━━━━━────────  2:13/4:24  🎧 AirPods Pro
```

| 你看到的 | 键 | 默认值 |
| --- | --- | --- |
| `▶︎` / `⏸` 状态图标 | 颜色跟随 `style.progressbar.playing` / `.paused` | `green` / `yellow` |
| `Karma Police` | `style.track.title` | `bold` |
| `— Radiohead` | `style.track.artist` | `italic` |
| `(Spotify)` | `style.app` | `dim` |
| `🔉` 音量图标 | `style.volume.icon` | `auto` |
| `▄` 音量条 | `style.volume.style`（形状） · `style.volume.bar`（显隐） | `block` · `on` |
| `45%` | `style.volume.percent` | `dim` |
| `━━━━━━━━━━━━────────` | `style.progressbar.style`（字符） · `style.progressbar.length`（格数） | `line` · `20` |
| `2:13` 已播时间 | `style.time.elapsed` | `bold` |
| `/4:24` 总时长 | `style.time.total` | `dim` |
| `🎧` 输出图标 | `style.output.icon` | `auto` |
| `AirPods Pro` | `style.output` | `dim` |

（哪些条目出现、排在哪一行，属于*布局*的事——见
[statusline.zh-CN.md](statusline.zh-CN.md)。）

整个组件共用**一种强调色**：▶︎/⏸ 图标、进度条的填充、音量条，播放时都用
`style.progressbar.playing` 的颜色，暂停时都用 `.paused` 的颜色。

## 进度条

字符由 `style.progressbar.style` 决定，宽度由 `style.progressbar.length`
决定（默认 20 格）。`/media:now` 回复里的进度条用同一套字符、同一个长度，
两处永远长得一样。字符和长度在关掉颜色后依然有效。

![进度条预设、音量条形状与十六进制强调色，以每秒一帧实时绘制](styles.gif)

`playhead` 不做填充：整条轨道保持细线，一个一格宽的粗游标以与 `glide`

```
 0%  ━───────────────────
47%  ─────────━──────────
50%  ─────────╼╾─────────
99%  ───────────────────━
```

### 动态预设

`wave` `pulse` `eq` `notes` 是**随长度自适应的波形**——形状由进度条宽度
算出，所以条越长，显示的波浪越大、越平滑，而不是把同样几个字形反复排列。
播放时缓缓漂移，暂停即静止。四种都铺满**整条**，用颜色标出进度。按长度
20 和 10 满条展示：

| 值 | 长度 20 | 长度 10 | |
| --- | --- | --- | --- |
| `wave` | `▁▃▅▇█▆▄▂▁▃▅▇█▆▄▂▁▃▅▇` | `▅▂▂▆█▅▂▂▆█` | 滚动起伏的正弦波 |
| `pulse` | `▁▁▁▁█▁▁▄▁▁▁▁▁▁█▁▁▄▁▁` | `▁▁█▁▁▄▁▁█▁` | 心电图波形——平坦基线、尖锐的 QRS、低矮的 T 波 |
| `eq` | `▆▄▅▄▄▇▆▅▄▂▂▄▅▅▆▄▄▅▅▇` | `▅▂▃▄▆▅▄▅▅▇` | 多频段均衡器 |
| `notes` | `··♪♫♪♫♪···♪♫♪♫♪···♪♫` | `♪··♫♪♫··♪♫` | ♪♫ 音符密度 |

每一种还有一个横向密度翻倍的**点阵孪生版**——`swell`（wave）、`bars`（eq）、
`ekg`（pulse）：

| 值 | 长度 20 | |
| --- | --- | --- |
| `swell` | `⣀⣴⣿⣿⣶⣄⠀⠀⣀⣴⣿⣿⣶⣄⠀⠀⣀⣴⣿⣿` | `wave` 的点阵版 |
| `bars` | `⣀⠀⣠⣦⣤⣶⣤⣠⣴⣶⣶⣶⣄⢀⣀⣀⣠⣶⣦⣤` | `eq` 的点阵版 |
| `ekg` | `⣀⣀⣀⣀⣀⣀⣀⣇⣀⣤⣀⣀⣀⣀⣀⣀⣀⣇⣀⣤` | 点阵版 `pulse`，细如针尖的心跳 |

### 音频可视化

四种预设把进度条变成频谱分析器。**块**版用八分块的高度（`▁`..`█`）；
**点阵**版每格塞进两条子列，横向密度翻倍。按长度 20 满条展示：

| 值 | 长度 20 | |
| --- | --- | --- |
| `spectrum` | `▂▂▆▅▄▆▆▃▂▅▆▄▅▆▄▂▅▆▅▄` | 各自独立的柱，偏重低音 |
| `mirror` | `▂▄▇█▆▃▁▂▅██▅▂▁▃▆█▇▄▂` | 以中心为轴镜像的波 |
| `cava` | `⢀⣦⣴⣆⢠⣦⣴⡀⣴⣤⣦⣠⣦⣤⣄⣴⣦⣤⣀⣶` | `spectrum` 的点阵版 |
| `ripple` | `⢀⣠⣾⣿⣦⣀⠀⢀⣴⣿⣿⣦⡀⠀⣀⣴⣿⣷⣄⡀` | `mirror` 的点阵版 |

波形和可视化器都铺满**整条**，而不只是已播放的部分。开着颜色时，强调色
与暗淡的分界标出进度；关掉颜色时，未播放的尾段会压低高度（`notes` 的尾段
只剩 `·` 休止符），所以光看形状也能读出进度。

### 心电图

`heartbeat` 和 `monitor` 画的也是心电图，只不过是绕着**居中的基线**上下
起伏，而不是从底部往上长——这换来了 `pulse` 和 `ekg` 做不到的那一下：
S 波会一路穿过基线，径直扎到线的*下方*。先走一段平线，隆起一个小小的
P 波，接着一根尖峰直冲顶端（R），随即一跌到底（S），收尾时再抬起一个
T 波。和上面那几对一样，它们也是一对孪生——`heartbeat` 用线条字符的竖笔
来画，`monitor` 用点阵描迹——而且共用同一个节拍，所以两者的跳动完全同步。

| 值 | 长度 20 | 长度 10 | |
| --- | --- | --- | --- |
| `heartbeat` | `━━━━┻┳━━━━━━━━┻┳━━━━` | `━━━━┻┳━━━━` | 线条字符的竖笔 |
| `monitor` | `⠤⠤⠤⠴⠼⡦⠤⠶⠤⠤⠤⠤⠤⠴⠼⡦⠤⠶⠤⠤` | `⠤⠤⠤⠴⠼⡦⠤⠶⠤⠤` | 点阵描迹 |

在所有波形里，只有这两个的形状**不**随进度条宽度变化——看上面两列就知道，
两种长度下心跳的大小完全一样。无论条有多长，心跳之间始终相隔 10 格，所以
条越长，显示的心跳次数越多，而不是把一次心跳拉长；长度 60 时，若让心跳随
长度自适应，两个波峰之间会空出 20 格左右的平线。

两者也和其余几种一样铺满整条，只是关掉颜色后，尾段会落回基线，而不是压低
高度——走成等电位线，就像真实描迹在两次心搏之间歇着那样：

```
heartbeat  40%  ━━━━┻┳━━━━━━━━━━━━━━
monitor    40%  ⠤⠤⠤⠴⠼⡦⠤⠶⠤⠤⠤⠤⠤⠤⠤⠤⠤⠤⠤⠤
```

`monitor` 有四行点阵可用，所以 P 波和 T 波都画得出来。线条字符每格只有
"有竖笔"和"没竖笔"两种状态，所以 `heartbeat` 只画得出那根尖峰——P 和 T
太小，撑不起一根竖笔。

### 精灵（sprite）

四种预设不填充进度条，而是在上面 **行走**。轨道走到哪儿，小动物就站在哪儿，
因此它的 **位置就是进度**——相当于把 `playhead` 的滑块换成了一个小动物。每种都走
在自己那条轨道上：走过的一半在身后，还没走的一半在前面。

按 34% 展示，取两帧中的第一帧：

| 取值 | 长度 20 | |
| --- | --- | --- |
| `cat` | `━━━━━━ᓚᘏᗢ┈┈┈┈┈┈┈┈┈┈┈` | 点线小路上的猫 |
| `snake` | `━━━━━━ᔓᔕᔓ╌╌╌╌╌╌╌╌╌╌╌` | 爬过虚线的蛇 |
| `duck` | `≈≈≈≈≈≈ᕬᗢ~~~~~~~~~~~~` | 水面上的鸭子 |
| `bird` | `━━━━━━⌒v⌒···········` | 点点星空下的鸟 |

腿的动作 **播放时每秒切换一次**，暂停时静止。多帧的意义正在于此：长度 20 时，
一首四分钟的曲子每 13 秒才走一格，而一只 13 秒才动一下的小动物，看着不像活的，
更像是坏了。所以无论猫有没有换格，每一拍都会换腿。下面它从 1:24 到 1:35 一直
原地踏步，然后才迈出一步：

```
1:24  ━━━━━━ᓚᘏᗢ┈┈┈┈┈┈┈┈┈┈┈      1:27  ━━━━━━ᓚᘐᗢ┈┈┈┈┈┈┈┈┈┈┈
1:25  ━━━━━━ᓚᘐᗢ┈┈┈┈┈┈┈┈┈┈┈       ...
1:26  ━━━━━━ᓚᘏᗢ┈┈┈┈┈┈┈┈┈┈┈      1:36  ━━━━━━━ᓚᘏᗢ┈┈┈┈┈┈┈┈┈┈
```

这四种是唯一完全不需要颜色的：光靠位置就能读出进度，所以关掉颜色后看起来
完全一样。开着颜色时，走过的轨道和它用强调色，前面的轨道则变暗淡。

sprite 是横跨多格的单个字形，所以它只占 **一个** seek 链接，而不是每格一个——
它脚下的那些格子没有各自的跳转目标。另外，窄到走不开的进度条（格数少于 sprite
的宽度）会改为按 `line` 绘制。

### 自定义精灵

`sprite` 就是这四种的"自带"入口，就像两字符写法之于那些填充预设一样。三个键：

| 键 | 含义 | 默认值 |
| --- | --- | --- |
| `style.progressbar.sprite` | 各帧，每秒切换一次 | `● ○` |
| `style.progressbar.trail` | 已经走过的轨道 | `━` |
| `style.progressbar.track` | 还没走的轨道 | `─` |

```
/media:config style.progressbar.style sprite            →  ━━━━━━━━━━●─────────
/media:config style.progressbar.sprite "ᓚᘏᗢ ᓚᘐᗢ"        →  ━━━━━━━━━ᓚᘏᗢ────────
/media:config style.progressbar.trail "═"
/media:config style.progressbar.track "┈"               →  ═════════ᓚᘏᗢ┈┈┈┈┈┈┈┈
```

各帧之间用空格分隔（所以帧本身不能含空格），并按写下的顺序循环，最多八帧。
**只有一帧时不会有动画** —— 一个只会走的标记，也就是字形随你挑的 `knob`。

```
/media:config style.progressbar.sprite "🚀"              →  ━━━━━━━━━🚀─────────
/media:config style.progressbar.sprite "▶"               →  ━━━━━━━━━━▶─────────
```

emoji 也可以用。它占两列宽，而进度条已经把这一点算进去了，所以画出来仍然正好是
`style.progressbar.length` 那么宽。但 `trail` 和 `track` 的字形必须各占 **一列**
（宽字形会让每格画两列，进度条就超出去了）；空格也允许，那一半就是隐形的。

上面那几种现成的 sprite 仍保留自己的帧和轨道：这几个键只作用于 `sprite`，就像
自己指定的 `"#-"` 两个字符不会影响 `blocks` 一样。

### 自定义字符

给**恰好两个字符**就是"填充 + 空白"（空白那格可以用空格；两个空格、
制表符和换行会被拒绝）：

```
/media:config style.progressbar.style "#-"     →  ############--------
/media:config style.progressbar.style "~ "     →  ~~~~~~~~~~~~
```

### 进度条长度

`style.progressbar.length` 决定进度条占多少格——1 到 60 之间的整数，
默认 `20`：

```
/media:config style.progressbar.length 10   →  ━━━━━━────
/media:config style.progressbar.length 40   →  ━━━━━━━━━━━━━━━━━━━━━━━━────────────────
```

一个长度同时管两处——状态栏组件和 `/media:now` 的回复。链接开着时每一格
照样可以 ⌘+点击，条越长，跳转的步进就越细。（音量迷你条每级音量一格，
固定 8 格。）默认值在 0.20.0 从 10 格加宽到 20 格——设回 `10` 就是
从前那根紧凑的条。

### 进度条颜色

`style.progressbar.playing`（默认 `green`）和 `.paused`（默认 `yellow`）
决定填充的颜色——组件共用一种强调色，所以 ▶︎/⏸ 图标和音量条会一起变。
空格子始终保持暗淡（dim）。

```
/media:config style.progressbar.playing bright-cyan
/media:config style.progressbar.playing "#3ddc84"   # 任何十六进制颜色都行
/media:config style.progressbar.paused magenta
```

## 音量

`volume` 条目按**图标 + 音量条 + 百分比**（`🔉 ▄ 45%`）渲染，静音时收拢成
一个 `🔇`。（需要 native helper——见 `/media:doctor`。）

### 音量条形状 — `style.volume.style`

| 值 | 10% | 20% | 35% | 50% | 60% | 75% | 85% | 100% | |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `block` | `▁` | `▂` | `▃` | `▄` | `▅` | `▆` | `▇` | `█` | 单格，高度随音量（默认） |
| `stairs` | `▁` | `▁▂` | `▁▂▃` | `▁▂▃▄` | `▁▂▃▄▅` | `▁▂▃▄▅▆` | `▁▂▃▄▅▆▇` | `▁▂▃▄▅▆▇█` | 八级台阶 |
| `progress` | `━───────` | `━━──────` | `━━━─────` | `━━━━────` | `━━━━━───` | `━━━━━━──` | `━━━━━━━─` | `━━━━━━━━` | 8 格迷你条 |

`progress` 用你进度条的那套字符来画——连动画都一样。`blocks` 画出
`████░░░░`，`dots` 画出 `●●●●○○○○`。音量条的颜色永远跟随播放/暂停强调色；
`style.volume.bar off` 只隐藏这根条（`🔉 45%`）。

### 音量图标 — `style.volume.icon`

| 值 | 效果 |
| --- | --- |
| `auto`（默认） | `🔈` 低 · `🔉` 中 · `🔊` 高 · 为零时 `🔇` |
| `none` | 隐藏——`▄ 45%` |
| 任意字符，如 `♪` | `♪ ▄ 45%` |

静音时无论图标怎么设，都只显示 `🔇`。

### 百分比 — `style.volume.percent`

接受文本样式（默认 `dim`），设为 `off` 则去掉：`🔉 ▄`。

## 输出设备

`output` 条目是图标 + 设备名：`🎧 AirPods Pro`。

| 键 | 值 |
| --- | --- |
| `style.output.icon` | `auto`（默认）= 按设备类型：`🎧` 蓝牙和耳机孔 · `📺` HDMI/DisplayPort · `📶` AirPlay · `🔊` 扬声器——或 `none`，或任意字符 |
| `style.output` | 设备名的文本样式（默认 `dim`），`off` 则只留图标 |

## 文本样式

所有文本部分——曲名、歌手、应用、已播/总时长、音量百分比、输出设备名——
都接受**样式说明**：

- `bold`、`dim`、`italic`、`underline` 任意组合
- 颜色至多一种：`black` `red` `green` `yellow` `blue` `magenta` `cyan`
  `white`，`bright-<颜色>`（实际色调由你终端的调色板决定），或一个精确的
  十六进制色号——`#ff8800`，短写 `#f80`——按 24 位真彩色渲染
  （多数终端支持；Apple Terminal 不支持）
- 或 `none`——完全不加样式
- 或 `off`——**隐藏这一部分**

```
/media:config style.track.title "bold bright-cyan"
/media:config style.track.title "bold #ff8800"   # 精确颜色——hex 要加引号
/media:config style.track.artist off
```

隐藏会顺手收拾干净：隐藏曲名，`—` 分隔符跟着消失；隐藏已播时间，总时长
前面的 `/` 也消失；一个条目的所有部分都被隐藏，整个条目就不再出现。
（想去掉整个条目，改布局才是正路——`/media:statusline`。）

样式说明只在 `statusline.color` 开着时渲染（`NO_COLOR` 永远优先）。改字符
的那些——进度条字符、音量条形状、图标——以及 `off`，不管颜色开没开都生效。
